# PRD Clarification Question Bank

Ask **5–10** questions max. Prefer multiple-choice or short answers.

## A) Audience & goal
1) Who is the primary user? (choose one)
   - Consumer / Individual
   - Small team
   - Company / Ops
   - Developer / Technical user
2) What is the #1 problem this solves (one sentence)?
3) What does success look like for the user after using it once?

## B) Scope
4) What are the must-have features for MVP (top 3–5)?
5) What is explicitly out of scope for MVP?
6) Is this single-user, multi-user, or both?

## C) Platforms & constraints (not tech stack)
7) Where will users access it?
   - Web
   - Mobile
   - Desktop
   - Mixed
8) Any constraints from the environment? (offline, low bandwidth, regulated data)
9) Any hard limits? (e.g., grid size, response time, max records, file sizes)

## D) Data & workflows
10) What core “things” does the system track? (entities)
11) What are the key workflows users repeat weekly/daily?
12) Do users import/export data? If yes, formats?

## E) Permissions & safety
13) Roles: who can view/edit/delete?
14) Any sensitive actions requiring confirmation or audit trail?

## F) Quality attributes
15) Performance expectations (what should feel instant vs can take time)?
16) Accessibility expectations (keyboard-only, screen readers, color contrast)?
17) Reliability expectations (how bad is downtime/data loss)?

## G) Analytics & success metrics
18) What metrics matter most? (choose 2–4)
   - Activation
   - Engagement
   - Retention
   - Conversion
   - Time saved
   - Error reduction
   - Revenue

## H) Assumptions policy
If the user can’t answer:
- Propose 2 reasonable options and pick one as an **assumption**.
- List the alternative as an **open question**.
